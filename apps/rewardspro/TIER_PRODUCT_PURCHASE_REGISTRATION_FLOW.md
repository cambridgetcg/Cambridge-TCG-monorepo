# Tier Product Purchase Registration - Complete Flow

**Date**: January 2025
**Purpose**: Explain how a tier product purchase gets registered from Shopify order to database

---

## 🎯 Overview

When a customer purchases a tier membership product in Shopify, the system automatically:
1. Receives webhook notification from Shopify
2. Identifies tier product in order line items
3. Creates TierPurchase database record
4. Resolves effective tier using 4-level priority system
5. Assigns tier to customer

## 🔍 Key Matching Logic

### How Tier Products Are Identified in Orders

**3 Ways to Match** (Lines 313-323):

```typescript
const tierProduct = await db.tierProduct.findFirst({
  where: {
    shop,
    OR: [
      { shopifyProductId: lineItem.product_id?.toString() },   // Method 1
      { shopifyVariantId: lineItem.variant_id?.toString() },   // Method 2
      { sku: lineItem.sku }                                    // Method 3
    ],
    purchaseType: { in: ['ONE_TIME', 'BOTH'] }
  }
});
```

**Priority**:
1. **Product ID match** - Most reliable (product level)
2. **Variant ID match** - Specific variant
3. **SKU match** - Fallback if IDs missing

**Why 3 Methods?**
- Product ID: Matches any variant of the product
- Variant ID: Matches exact variant (if multiple variants exist)
- SKU: Backup if Shopify IDs change or missing

**Example**:
```
TierProduct in DB:
- shopifyProductId: "8123456789"
- shopifyVariantId: "43123456789"
- sku: "tier-gold-annual"

Order Line Item:
- product_id: 8123456789        ✅ Match!
- variant_id: 43123456789       ✅ Match!
- sku: "tier-gold-annual"       ✅ Match!

Result: Tier product identified
```

## 🎯 Important Notes

### ✅ What Happens Automatically
1. Webhook received and verified
2. Order stored in database
3. Tier product identified by product/variant/SKU
4. TierPurchase record created with expiration
5. Tier resolution checks all 4 sources
6. Customer tier assigned (highest priority wins)
7. Tier change logged for audit
8. Cashback calculated (if applicable)
9. Spending totals updated

### ⚠️ What Does NOT Happen
1. **No automatic store credit** - Cashback is PENDING, requires manual approval
2. **No tier downgrade protection** - When purchase expires, tier may downgrade
3. **No email notification** - Must be implemented separately
4. **No Shopify customer metafield update** - Tier is only in our database

### 🔐 Security Features
1. **HMAC verification** - Ensures webhook is from Shopify
2. **Idempotency** - Prevents duplicate processing if webhook retries
3. **Shop isolation** - All queries scoped to authenticated shop
4. **Price validation** - Sanitizes and validates prices
5. **Transaction safety** - Uses database transactions where needed

### 🔄 Tier Resolution Priority (CRITICAL)
```
Priority 1: Manual Override      (Admin sets tier manually)
   ↓
Priority 2: Tier Subscription    (Recurring billing, active)
   ↓
Priority 3: Tier Purchase        (One-time purchase, active & not expired) ← THIS FLOW
   ↓
Priority 4: Spending-Based       (Calculated from order history)
```

**Example Scenario**:
- Customer has Gold purchase (Priority 3)
- Customer also qualifies for Silver by spending (Priority 4)
- **Result**: Customer gets Gold (Priority 3 wins)

**Edge Case**:
- Customer has Gold purchase (expiring tomorrow)
- Customer also has active Bronze subscription (Priority 2)
- **Result**: Customer gets Bronze (Priority 2 wins over 3)

### 📊 Status Flow
```
ACTIVE → Customer has access to tier
   ↓
EXPIRED → endDate passed, tier resolution will reassign
   ↓
REFUNDED → Order refunded, purchase invalidated
```

## 🐛 Troubleshooting

### Purchase Not Registered

**Symptom**: Customer bought tier product but no TierPurchase record

**Check**:
1. **Is TierProduct in database?**
   ```sql
   SELECT * FROM TierProduct
   WHERE shopifyProductId = 'product-id'
   OR shopifyVariantId = 'variant-id'
   OR sku = 'product-sku';
   ```

2. **Did webhook fire?**
   - Check webhook logs in Shopify admin
   - Check WebhookProcessed table

3. **Was HMAC valid?**
   - Check app logs for HMAC errors
   - Verify SHOPIFY_WEBHOOK_SECRET is correct

4. **Match criteria met?**
   - Product ID, Variant ID, or SKU must match
   - purchaseType must be ONE_TIME or BOTH

### Tier Not Assigned

**Symptom**: TierPurchase exists but customer has different tier

**Check**:
1. **Tier resolution priority**
   - Customer might have higher priority tier source
   - Check: Manual Override > Subscription > Purchase > Spending

2. **Purchase status**
   ```sql
   SELECT status, endDate
   FROM TierPurchase
   WHERE id = 'purchase-id';
   ```
   - Must be ACTIVE
   - endDate must be null or in future

3. **Run resolution manually**
   - Use "Recalculate Tier" button in customer page
   - Check logs to see which source won

### Purchase Shows as Expired

**Symptom**: Purchase created but immediately expired

**Check**:
1. **Duration calculation**
   - MONTHLY → +1 month
   - ANNUAL → +1 year
   - LIFETIME → null endDate

2. **Timezone issues**
   - Dates stored in UTC
   - Check if endDate < NOW() in UTC

3. **Status field**
   - Should be 'ACTIVE' not 'EXPIRED'
