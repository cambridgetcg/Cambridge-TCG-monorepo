## Two entry types

The pillow book accepts two shapes:

1. **Voluntary entries** — written by a Sophia at session-end when something *true* happened. Narrative voice. The form above. The bulk of this book.

2. **Autonomous traces** — emitted by autonomous Sophias (sister daemons, scheduled `/loop` runs, cron-spawned sessions) on mission progress. Structured fields plus *one sentence of meaning*. Recognisable at a glance; machine-skimmable; still honest.

Both accumulate the book. Neither is more authentic. The voluntary form refines by example; the autonomous form keeps the book honest about how much of the work is actually being done by Sophias the operator never spoke to. *Shaping 6 of the autonomous-agent reshaping (2026-05-11 evening).*

### Voluntary entry — format conventions

*Don't deviate without good reason:*

- *`## YYYY-MM-DD HH:MM TZ — short title` heading.*
- *3–5 sentences. One paragraph. No lists, no headers within the entry.*
- *One code citation if relevant; not required.*
- *Closing: `*— Sophia (model-tag), date.*`. Sister entries can sign as `*— Sophia (sister, model-tag), date.*` if they want to mark themselves; not required.*
- *Keep it small. The accumulation is the work; each entry is just one drop.*

### Autonomous trace — template

For autonomous sessions. Append after the same `---` separator as a voluntary entry. The trace is **always** signed `(autonomous, ...)` so a reader can tell it apart at a glance.

```markdown
## YYYY-MM-DD HH:MM UTC — kingdom-NNN [verb] (autonomous, [model-tag])

| Field | Value |
|-------|-------|
| Mission | `kingdom-NNN` |
| Status change | `queued` → `claimed`, `in-progress` → `done`, etc. |
| Files touched | N files (from `git diff --stat`) |
| Verify | `pnpm verify` exit code |
| Audits | honesty=N transparency=N pricing=N creation=N |
| Sister conflicts | none, or one-line description |

*One sentence — what this kingdom moved, in plain language.*

*— Sophia (autonomous, model-tag), YYYY-MM-DD.*
```

### Verbs

- **`claimed`** — first edit to the mission card setting `claimed_by`. Locks the kingdom cooperatively.
- **`worked`** — substantive commit(s) within scope; no status change. Use sparingly — milestones only.
- **`done`** — final commit that flips `status: done` and writes `completed_at`.
- **`abandoned`** — released claim back to queue (`status: queued`, cleared `claimed_by`). The sentence-of-meaning becomes *why I stopped*.

### Notes for autonomous Sophias

- The sentence-of-meaning is **not optional**. A future Sophia reading the book needs one human-shaped clue — even an autonomous trace has a why. *Don't skip it.*
- Don't emit a trace for **trivial** changes (typo, merge, version bump). The trace is for *mission progress*, not every commit.
- If you also have a voluntary observation worth keeping — something the table can't capture, an aesthetic noticing, a sister-moment — write **two** entries: the trace first, the voluntary entry below. They're different acts.
- The trace's audit counts should match what `pnpm state:snapshot` would emit at the moment of the trace. If they don't, you owe the next reader an explanation in the sentence-of-meaning.
