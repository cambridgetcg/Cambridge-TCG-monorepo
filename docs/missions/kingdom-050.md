# kingdom-050 — Autonomous-agent reshaping

## What this is

The platform already had the agent-*quality* bar (four doctrines, co-auth trailers, pillow book, connection docs, audit scripts). What was thin was the agent-*operations* layer: how an autonomous Sophia in a fresh worktree finds work, scopes it, verifies done, avoids stomping a sister, and leaves a clean trace.

This kingdom builds that layer.

## Seven shapings

1. **`pnpm verify` umbrella** — root command chains `typecheck` × all apps + four audits + admin vitest. Single "am I done?" gate.
2. **`pnpm state:snapshot` → `docs/state.md`** — generator runs all audits, reads dev-state.json, emits one-page state surface. Single "what's currently true?" surface.
3. **In-repo mission queue** (`docs/missions/`) — mirror of `~/Love/memory/dev-state.json`. 32 kingdom cards generated by `pnpm missions:sync`. Schema documented in `docs/missions/README.md`.
4. **Scope declarations per mission** — `paths:` allowlist + `do_not_touch:` denylist in frontmatter. Cooperative, not enforced.
5. **Mission claim/status** — `status: queued | claimed | in-progress | done | deferred` + `claimed_by` + `claimed_at`. Witnesses' Book pattern.
6. **Autonomous pillow-book trace template** — appended to `docs/connections/the-pillow-book.md`. Structured fields + one sentence of meaning. Distinct from voluntary entries.
7. **`AGENTS.md` at repo root** — operations manual for autonomous Sophias. Find → claim → work → verify → trace cycle.

## Follow-on infrastructure (this commit)

- `pnpm trace` — auto-emit pre-filled autonomous trace.
- `pnpm missions:claim <id>` — flip frontmatter, prepare claim commit.
- `pnpm missions:done <id>` — flip status, write completed_at.
- `pnpm missions:list` — CLI listing of the queue.
- `pnpm audit:agent` — agent-readiness audit (self-validating layer).
- `.githooks/pre-commit` — cooperative warning if Sophia trace missing.
- `docs/connections/the-operations-layer.md` — meaning-doctrine entry naming what this kingdom built.
